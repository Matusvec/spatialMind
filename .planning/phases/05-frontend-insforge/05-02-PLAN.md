```typescript
interface SparkRendererProps {
  active: boolean;

  heatmapQuery?: string | null;    // When set, fetch probability cloud and apply
  heatmapEnabled?: boolean;        // Toggle heatmap visualization
}
```

2. Change SPLAT_URL from hardcoded localhost:3000 to `${API_URL}/api/scene/splat`.

3. Store a ref to the SplatMesh instance: `const splatRef = useRef<any>(null);`

4. Add a useEffect that watches `highlightQuery`. When it changes and is non-null:
   - Fetch `${API_URL}/api/clip/highlight` with POST body `{text: highlightQuery, k: 200, mode: "soft"}`
   - On response, iterate the mask array. For indices where mask[i] > 0.3, call `splatRef.current.dyno.setColor(i, 1.0, 0.85, 0.0, mask[i])` (golden yellow highlight, alpha from mask).
   - For indices where mask[i] <= 0.3, call `splatRef.current.dyno.setColor(i, 0.3, 0.3, 0.3, 0.15)` (dim non-matching Gaussians).
   - When highlightQuery becomes null, call `splatRef.current.dyno.resetAllColors()`.
   - Wrap in try/catch. If the dyno API is not available (older Spark.js), log warning and skip.
   - IMPORTANT: The mask array may have 100K+ entries. Process in batches of 10000 using requestAnimationFrame to avoid blocking the main thread.

5. Add a useEffect that watches `heatmapQuery` + `heatmapEnabled`. When both are truthy:
   - Fetch `${API_URL}/api/clip/probability` with POST body `{text: heatmapQuery, temperature: 0.1}`
   - Map probabilities to a blue-to-red color ramp:
     - score < 0.2: blue (0.2, 0.3, 0.8) at low opacity
     - 0.2-0.5: cyan (0.2, 0.7, 0.8)
     - 0.5-0.7: yellow (0.9, 0.8, 0.2)
     - score > 0.7: red (1.0, 0.3, 0.1)
     - Alpha = score (continuous gradient)
   - Apply via dyno.setColor same as highlight.
   - When heatmapEnabled becomes false, resetAllColors.

6. Add OrbitControls for camera interaction. Import from three/addons:

```typescript
import { OrbitControls } from 'three/addons/controls/OrbitControls.js';
```

```typescript
controls.target.set(0, 0.5, 0);
```
Call `controls.update()` in the animation loop.

7. In the xhr.onload callback, store the SplatMesh ref: `splatRef.current = splat;`
