Create room-cognito/src/components/GraphOverlay3D.tsx.

This component renders 3D graph nodes and edges as Three.js objects overlaid on the splat scene. It receives a Three.js scene ref and adds/removes its objects.

```typescript
interface GraphOverlay3DProps {
  scene: THREE.Scene | null;        // Shared Three.js scene from SparkRenderer
  objects: DetectedObject[];         // Objects with 3D positions
  graphNodes: GraphNode[];           // Graph node metadata
  graphEdges: GraphEdge[];           // Graph edges
  visible: boolean;                  // Toggle overlay visibility
  highlightedObjects?: string[];     // Labels to highlight
}
```

Implementation approach:
1. Use a `useEffect` that watches objects, graphEdges, visible. Create a THREE.Group called "graphOverlay".

2. For each DetectedObject, create:
   - A THREE.Mesh with SphereGeometry(0.05, 8, 8) at the object's position [x, y, z]
   - Color: #6366f1 (indigo) for normal, #fbbf24 (gold) for highlighted
   - Material: MeshBasicMaterial with transparent=true, opacity=0.8
   - A THREE.Sprite with SpriteMaterial for the label text. Use CanvasTexture:
     - Create an offscreen canvas (256x64)
     - Draw white text on transparent background: ctx.fillStyle = "#ffffff", ctx.font = "bold 24px Inter, sans-serif"
     - Create SpriteMaterial with map: new THREE.CanvasTexture(canvas), transparent: true
     - Position sprite slightly above the sphere (y + 0.12)
     - Scale sprite to (0.3, 0.08, 1)

3. For each GraphEdge, find the source and target positions by matching edge.from/edge.to to node IDs, then to DetectedObject positions (match by label or ID). Create:
   - A THREE.Line with BufferGeometry containing two points (source position, target position)
   - Material: LineBasicMaterial with color=#6366f1, opacity=0.4, transparent=true

4. Add the group to the scene. On cleanup (useEffect return), remove the group from scene and dispose all geometries/materials.

5. When `visible` is false, set group.visible = false (don't remove, just hide).

6. When `highlightedObjects` changes, update sphere colors: gold for highlighted, indigo for normal.

7. Use useRef to store the group and avoid recreating on every render. Only recreate when objects/edges change.

Export the component as a named export.

IMPORTANT: The positions from the backend are in the splat's coordinate system. They should map naturally to Three.js coordinates since both use a right-handed Y-up system. If objects appear in wrong positions, the adapter can be adjusted later -- for now use positions as-is.
