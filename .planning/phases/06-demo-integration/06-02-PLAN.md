The script must run these checks in order, printing PASS/FAIL for each:

**Pre-check: Pipeline readiness**
- GET `{base}/api/health`
- Assert response contains `"status": "ok"` and `"pipeline_ready": true`
- Print: "Pipeline: {gaussian_count} Gaussians, scene_source={scene_source}"
- If pipeline_ready is false, print which components are missing and exit(1).

**Beat 1: Object highlighting ("chairs")**
- POST `{base}/api/clip/highlight` with body `{"text": "chairs", "k": 100}`
- Assert response contains `"indices"` key with a non-empty list
- Assert response contains `"scores"` key with a non-empty list
- Assert `len(indices) > 0` (at least some Gaussians matched)
- Print: "Beat 1 PASS: {len(indices)} Gaussians highlighted for 'chairs'"
- Measure and print response time: "Response time: {ms}ms"

**Beat 2: Spatial reasoning ("what's on the tables?")**
- POST `{base}/api/query` with body `{"query": "what's on the tables?", "session_id": "e2e-test"}`
- Assert response contains `"answer"` or `"description"` key with a non-empty string
- Assert the answer string length is > 10 characters (not a stub)
- Print: "Beat 2 PASS: Got answer ({len} chars): '{first_80_chars}...'"
- Measure and print response time.

**Beat 3: Unrehearsed query ("where are the windows?")**
- POST `{base}/api/query` with body `{"query": "where are the windows?", "session_id": "e2e-test"}`
- Assert response contains `"answer"` or `"description"` key with a non-empty string (> 10 chars)
- Print: "Beat 3 PASS: Got answer ({len} chars): '{first_80_chars}...'"
- Measure and print response time.

**Summary:**
- Print total pass/fail count.
- Print: "DEMO READY" if all 3 pass, or "DEMO NOT READY - {N} beats failed" with details.
- Exit with code 0 if all pass, 1 if any fail.

Each beat should have a 30-second timeout. Use `urllib.request.urlopen` with a timeout parameter. Wrap each beat in try/except to handle connection errors gracefully (print FAIL, continue to next beat).
