<objective>
Build the core semantic query pipeline: FastAPI server that loads LangSplat artifacts (PLY + autoencoder), encodes text queries via CLIP, decodes Gaussian latents to 512-dim CLIP space, computes cosine similarity, and returns top-K matching Gaussian indices.

Purpose: This is the computational backbone of SpatialMind. Every downstream feature (highlighting, probability clouds, graph construction, walker queries) depends on this CLIP-to-Gaussian similarity pipeline working correctly.

Output: A running FastAPI server on port 8001 with `/health` and `/clip/encode` endpoints that accept text and return matching Gaussian indices with similarity scores.
</objective>

Autoencoder architecture (from STACK.md research):
- Encoder dims: [256, 128, 64, 32, 3]
- Decoder dims: [16, 32, 64, 128, 256, 256, 512]
- Input to decoder: 3-dim latent
- Output from decoder: 512-dim CLIP-space vector

Artifacts directory structure (from Phase 1):
```
<scene_dir>/artifacts/
  point_cloud.ply       # Gaussian positions + covariances + language_feature (3-dim latent)
  autoencoder.pth       # Scene-specific decoder: 3-dim -> 512-dim
  language_feature_dim3/
    *.npy               # Per-image compressed features (used in training, NOT needed at query time)
```

Note: At query time, the 3-dim latent features are stored as properties IN the PLY file (per-Gaussian), not as separate .npy files. The .npy files are per-image training intermediates. The PLY vertex properties include language_feature_0, language_feature_1, language_feature_2.

    **server/models/autoencoder.py:**
    Define `class Autoencoder(nn.Module)` matching LangSplat's architecture exactly:
    - Encoder: Linear layers with dims input_dim -> 256 -> 128 -> 64 -> 32 -> 3, each followed by ReLU except the last
    - Decoder: Linear layers with dims 3 -> 16 -> 32 -> 64 -> 128 -> 256 -> 256 -> 512, each followed by ReLU except the last
    - Methods: `encode(x) -> Tensor`, `decode(x) -> Tensor`, `forward(x) -> Tensor` (encode then decode)

    CRITICAL: The autoencoder saved by LangSplat's training script may be saved as a full model object (torch.save(model)) not just state_dict. Support BOTH loading patterns:
    1. Try `torch.load(path, map_location=device)` — if result has `.decode()` method, use directly
    2. If result is a state_dict (OrderedDict), instantiate Autoencoder() and load state_dict

    **server/services/gaussian_store.py:**
    Define `class GaussianStore`:
    - `__init__(self, config)` — stores config reference
    - `load_ply(self, ply_path: str)` — uses plyfile to read PLY, extracts:
      - `self.positions: np.ndarray` shape (N, 3) from vertex x, y, z
      - `self.latent_features: np.ndarray` shape (N, 3) from vertex language_feature_0, language_feature_1, language_feature_2
      - `self.count: int` = N (number of Gaussians)
      - If language_feature_* properties not found, try f_language_feature_0 etc. (LangSplat naming varies)
    - `decode_all(self, autoencoder, device)` — decode ALL latent features to 512-dim at startup:
      - Convert latent_features to torch tensor
      - Run autoencoder.decode() in batches of 10000 (memory safety)
      - L2-normalize each row
      - Store as `self.decoded_embeddings: np.ndarray` shape (N, 512) float32
    - `@property is_loaded(self) -> bool` — True if positions and decoded_embeddings are set

    **server/services/clip_encoder.py:**
    Define `class CLIPEncoder`:
    - `__init__(self, model_name, pretrained, device)` — loads open_clip model and tokenizer at init
      - `self.model, _, _ = open_clip.create_model_and_transforms(model_name, pretrained=pretrained)`
      - `self.tokenizer = open_clip.get_tokenizer(model_name)`
      - `self.model.eval().to(device)`
      - `self.device = device`
    - `encode_text(self, text: str) -> np.ndarray` — returns (512,) float32 normalized vector
      - Tokenize, encode, normalize, convert to numpy
      - Use `@functools.lru_cache(maxsize=256)` on the text string for caching (per Pitfall: CLIP encoding is 200-500ms on CPU)

    **server/services/similarity.py:**
    Define functions:
    - `def top_k(text_embedding: np.ndarray, gaussian_embeddings: np.ndarray, k: int = 100) -> tuple[np.ndarray, np.ndarray]`:
      - Compute cosine similarity: `similarities = gaussian_embeddings @ text_embedding` (both already L2-normalized)
      - Use `np.argpartition(-similarities, k)[:k]` for O(N) partial sort
      - Sort the top-k by descending similarity
      - Return `(indices, scores)` both as 1-D arrays
    - `def soft_scores(text_embedding: np.ndarray, gaussian_embeddings: np.ndarray) -> np.ndarray`:
      - Return raw cosine similarity for ALL Gaussians as (N,) float32 array
      - This is used by the probability cloud endpoint (Plan 02)

    **server/routers/clip.py:**
    Define Pydantic models:
    - `class EncodeRequest(BaseModel): text: str; k: int = 100`
    - `class EncodeResponse(BaseModel): query: str; k: int; indices: list[int]; scores: list[float]; gaussian_count: int`

    Endpoint:
    ```python
    @router.post("/clip/encode", response_model=EncodeResponse)
    async def encode_query(request: EncodeRequest):
        state = get_app_state()
        text_embedding = state["clip_encoder"].encode_text(request.text)
        indices, scores = top_k(text_embedding, state["gaussian_store"].decoded_embeddings, k=request.k)
        return EncodeResponse(
            query=request.text,
            k=request.k,
            indices=indices.tolist(),
            scores=scores.tolist(),
            gaussian_count=state["gaussian_store"].count,
        )
    ```
